Getting Started met Git

Initiele Git Configuratie

  • Identiteit instellen 
    
 git config --global user.name "John Doe"
 git config --global user.email johndoe@example.com
  • Aanpassen van editor 
    • 
git config --global core.editor 'c:/program files (x86)/brackets/brackets.exe'

Een repo aanmaken

  • Vanuit een bestaande map 
    
cd /home/user/myprojdir
git init
git remote add <naam_remote> <link_naar_remote>
# de remote toevoegen is optioneel, je kan ook lokaal werken
  • Een bestaande repo gebruiken 
    
git clone https://github.com/luytsm/mcu-cursus
git clone git@github.com:luytsm/mcu-cursus.git
# Je kan zowel een repo clonen over https als over ssh verbinding

Toevoegen van bestanden

Verschillende stages van een file

Staging Simple

Staging

Toevoegen van bestanden

  • Aanmaken van een README bestand
    • 
echo 'My Project' > README
git status
On branch master
Your branch is up-to-date with 'origin/master'.
Untracked files:
  (use "git add <file>..." to include in what will be commited)

    README

nothing added to commit but untracked files present (use "git add" to track)
  • Tracken van het README bestand
    • 
git add README
If you run your status command again, you can see that your README file is now tracked and staged to be committed:

$ git status
On branch master
Your branch is up-to-date with 'origin/master'.
Changes to be committed:
  (use "git reset HEAD <file>..." to unstage)

    new file:   README

Toevoegen van bestanden

  • Commiten van het README bestand
    • 
git commit
  • Schrijven van commit message
    • Kort commit bericht
    • Weergave van aanpassingen
    • Lang commit bericht
    
# Please enter the commit message for your changes. Lines starting
# with '#' will be ignored, and an empty message aborts the commit.
# On branch master
# Your branch is up-to-date with 'origin/master'.
#
# Changes to be committed:
#   new file:   README
~
".git/COMMIT_EDITMSG" 9L, 283C
  • Aanpassen van editor 
    • 
git config --global core.editor 'c:/program files (x86)/brackets/brackets.exe'

Verwijderen / verplaatsen van bestanden

  • Verwijderen van bestanden
    • 
git rm README
# Je moet deze verandering ook commiten
  • Verplaasten van bestanden
    • 
mv README README.md
git rm README
git add README.md
# de lange manier
# Je moet deze verandering ook commiten
    
    
git mv README README.md
# de korte manier
# Je moet deze verandering ook commiten

Terugzetten van files

  • Commit aanpassen
    • 
git commit -m 'initial commit'
git add forgotten_file
git commit --amend
  • Een item niet meer stagen
    • 
# Een aanpassing die klaar staat om te commiten weghalen
# Wijzigingen in de file blijven behouden
git reset README.md
  • Een file wijzigen naar de vorige state
    • 
# Een aangepaste file die terugzetten naar de laatst gecommite versie
git checkout README.md

Werken met remotes

Een remote is een van de peers die een kopie bevat van de repo
  • Tonen remote van een kloon
    • 
git clone https://github.com/luytsm/dops-cursus
git remove -v

#output
origin  git@github.com:luytsm/dops-cursus.git (fetch)
origin  git@github.com:luytsm/dops-cursus.git (fetch)
  • Changes ophalen van een remote
    • 
git pull <remote_naam> <branch_naam>
  • Changes verzenden naar een remote
    • 
git push <remote_naam> <branch_naam>
  • Standaard manier om met remotes te werken
    • 
git commit
git pull
git push
    • Lokaal alles committen
    • Alles ophallen
    • Eventueel mergen
    • Alles ophalen

Branches

  • Branch aanmaken
    • 
git branch newbranch
  • Branch uitchecken
    • 
git checkout newbranch
  • Oneliner
    • 
git checkout -b newbranch
